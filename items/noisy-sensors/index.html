Overview

Every physical sensor — encoder, IMU, force/torque sensor, camera — introduces error into its readings. This error is called measurement noise, and understanding its properties is the foundation of all state estimation.

In this module, a simple pendulum swings back and forth. The "encoder" attached to its pivot measures the angle $q(t)$, but every reading $y(t)$ is corrupted by additive Gaussian noise $v(t)$. Use the sliders above to see how noise magnitude ($\sigma$) affects the quality of measurements.

Key Insight

The signal-to-noise ratio (SNR) determines how much useful information each measurement contains. When SNR drops below ~10 dB, raw measurements become unreliable and filtering becomes essential.

When to Use

  • Characterizing a new sensor's noise properties
  • Deciding whether raw sensor data is "good enough" or needs filtering
  • Setting the measurement noise covariance $R$ in a Kalman filter
  • Comparing sensor quality across different hardware

Key Properties

  • Additive — noise adds to the true signal: $y = q + v$
  • Zero-mean — on average, noise doesn't bias the measurement: $\mathbb{E}[v] = 0$
  • Gaussian — noise follows a normal distribution: $v \sim \mathcal{N}(0, \sigma^2)$
  • White — noise at different times is uncorrelated: $\mathbb{E}[v(t_1)v(t_2)] = 0$ for $t_1 \neq t_2$

Advantages of This Model

  • Mathematically tractable
  • Good first-order approximation for many sensors
  • Central Limit Theorem justifies Gaussian assumption
  • Single parameter ($\sigma$) fully characterizes noise

Limitations

  • Real noise may be non-Gaussian (outliers, bias drift)
  • Doesn't capture colored (correlated) noise
  • Ignores quantization effects
  • May underestimate tail events

How It Works

The interactive diagram above simulates a pendulum with a noisy encoder. Here's what's happening:

  1. True Signal: The pendulum angle follows $q(t) = A\sin(2\pi f t)$, shown as the smooth green curve. This is what we want to know.
  2. Noise Generation: At each sample time $t_k$, an independent random value $v_k \sim \mathcal{N}(0, \sigma^2)$ is drawn using the Box-Muller transform.
  3. Measurement: The sensor reports $y_k = q(t_k) + v_k$. These are the yellow dots scattered around the true curve.
  4. Residuals: The difference $r_k = y_k - q(t_k) = v_k$ should form a Gaussian distribution (toggle "Show Residuals" to verify).

Try increasing $\sigma$ and watch the SNR drop. Notice how the measurements become less and less useful as noise overwhelms the signal.

Key Equations

Measurement Model

$$y_k = q(t_k) + v_k, \quad v_k \sim \mathcal{N}(0, \sigma^2)$$

Gaussian Probability Density

$$p(v) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{v^2}{2\sigma^2}\right)$$

Signal-to-Noise Ratio (dB)

$$\text{SNR}_{\text{dB}} = 10 \log_{10}\!\left(\frac{P_{\text{signal}}}{P_{\text{noise}}}\right) = 10 \log_{10}\!\left(\frac{A^2/2}{\sigma^2}\right)$$

Measurement Covariance (Scalar)

$$R = \mathbb{E}[v_k^2] = \sigma^2$$

Sample Variance Estimator

$$\hat{\sigma}^2 = \frac{1}{N-1}\sum_{k=1}^{N}(y_k - \bar{y})^2$$

MATLAB Implementation

MATLAB
%% E01: Noisy Sensors — Pendulum Measurement Simulation
% Simulate a pendulum with additive Gaussian sensor noise

% Parameters
A     = 1.0;        % amplitude (rad)
f     = 0.5;        % frequency (Hz)
sigma = 0.15;       % noise std dev (rad)
dt    = 0.02;       % sample period (s)
T     = 6.0;        % duration (s)

% Time vector
t = 0:dt:T;
N = length(t);

% True signal
q_true = A * sin(2*pi*f*t);

% Noisy measurements
v = sigma * randn(1, N);    % Gaussian noise
y = q_true + v;             % measurement = truth + noise

% Compute SNR
P_signal = mean(q_true.^2);
P_noise  = mean(v.^2);
SNR_dB   = 10*log10(P_signal / P_noise);

fprintf('SNR = %.1f dB\n', SNR_dB);
fprintf('Empirical sigma = %.4f (true = %.4f)\n', std(v), sigma);

% Plot
figure;
subplot(2,1,1);
plot(t, q_true, 'g-', 'LineWidth', 2); hold on;
plot(t, y, 'y.', 'MarkerSize', 4);
legend('True q(t)', 'Measurements y_k');
xlabel('Time (s)'); ylabel('Angle (rad)');
title(sprintf('Noisy Pendulum (SNR = %.1f dB)', SNR_dB));

subplot(2,1,2);
histogram(v, 30, 'Normalization', 'pdf');
hold on;
vv = linspace(-3*sigma, 3*sigma, 100);
plot(vv, normpdf(vv, 0, sigma), 'r-', 'LineWidth', 2);
xlabel('Residual (rad)'); ylabel('PDF');
title('Noise Distribution');

Python Implementation

Python (NumPy)
import numpy as np
import matplotlib.pyplot as plt

# Parameters
A, f, sigma = 1.0, 0.5, 0.15
dt, T = 0.02, 6.0
t = np.arange(0, T + dt, dt)

# True signal & noisy measurements
q_true = A * np.sin(2 * np.pi * f * t)
v = sigma * np.random.randn(len(t))
y = q_true + v

# SNR
snr_db = 10 * np.log10(np.mean(q_true**2) / np.mean(v**2))
print(f"SNR = {snr_db:.1f} dB")

# Plot
fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(10, 6))
ax1.plot(t, q_true, 'g-', lw=2, label='True q(t)')
ax1.scatter(t, y, c='gold', s=4, alpha=0.5, label='Measurements')
ax1.legend(); ax1.set_xlabel('Time (s)'); ax1.set_ylabel('Angle (rad)')
ax2.hist(v, bins=30, density=True, alpha=0.7, color='gold')
ax2.set_xlabel('Residual'); ax2.set_ylabel('PDF')
plt.tight_layout(); plt.show()
